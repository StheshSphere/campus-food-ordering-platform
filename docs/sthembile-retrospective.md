During Sprint 1 of the Campus Food Ordering Platform, I was responsible for implementing the admin controls functionality. My primary contribution was developing the admin dashboard that allows administrators to manage vendors. This included fetching vendor data from the Supabase database, displaying it in a structured table, and implementing functionality to approve or suspend vendors by updating their status. I also integrated the admin login flow so that users with the admin role are redirected to the admin dashboard after authentication. Additionally, I implemented role-based checks in the frontend to ensure that only admins can access and perform actions on this page.

One of the main challenges I encountered was working with relational data between the users and vendors tables. Specifically, I had to retrieve vendor information along with their associated user email addresses, which required careful handling of queries and mapping data correctly. Another difficulty was understanding how to properly enforce access control. Initially, I relied only on frontend checks, but I later learned that true security requires backend enforcement using Supabase Row Level Security policies.

If I were to approach this sprint again, I would focus earlier on designing a clear API/data interaction plan with my team. This would have reduced confusion when integrating different features and ensured consistency across our database interactions. I would also prioritise implementing backend security policies from the beginning instead of adding them later.

My work fits into the overall system by acting as the control layer for vendor access. Vendors cannot fully use the platform until approved by an admin, and they can be restricted if necessary. This ensures that the platform maintains quality and security, making the admin controls a critical part of the system’s workflow.
